miniCLI input syntax is intentionally compact and explicit.

## Positional resolution

```bash
./minicli <command> [subcommand]
```

## Named parameters

Use `key=value` for method parameters.

```bash
./minicli users create name=erika role=admin
```

## Flags

Boolean and global flags use `--flag` form.

```bash
./minicli users create --notify --profile
```

## Notes

- Parameter names map to kebab-case method parameter names unless overridden with `#[Argument(name: ...)]`.
- Unknown command names raise `CommandNotFoundException`.
